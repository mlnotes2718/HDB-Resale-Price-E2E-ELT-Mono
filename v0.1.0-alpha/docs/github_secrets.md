# GitHub Secrets Setup Guide

This guide provides step-by-step instructions on how to create and set GitHub Secrets for this HDB Resale Price E2E ELT project.

## Overview

GitHub Secrets are encrypted environment variables that are securely stored in GitHub. They are used to store sensitive information like API keys, authentication credentials, and service account keys that should not be exposed in code or logs.

This project requires **4 secrets** for the GitHub Actions workflow to run successfully:

1. `DBT_BIGQUERY_SERVICE_ACCOUNT_KEY` - BigQuery service account JSON key
2. `MAIL_USERNAME` - Gmail account email address
3. `MAIL_PASSWORD` - Gmail app password
4. `COLLABORATORS_EMAILS` - Email address(es) for workflow notifications

---

## Prerequisites

- A GitHub repository (this project)
- Contributor or Owner access to the repository
- GCP project with BigQuery API enabled and a service account key
- Gmail account with app password enabled

---

## Step 1: Prepare Your Secrets

Before creating secrets in GitHub, gather all the required information:

### 1.1 BigQuery Service Account Key (DBT_BIGQUERY_SERVICE_ACCOUNT_KEY)

**What it is**: A JSON file containing GCP service account credentials.

**How to get it**:
1. Go to [Google Cloud Console](https://console.cloud.google.com/)
2. Select your GCP project (must be under `No Organization`)
3. Navigate to **IAM & Admin** → **Service Accounts**
4. Create a new service account or select an existing one
5. Go to **Keys** → **Add Key** → **Create new key**
6. Choose **JSON** format and click **Create**
7. A JSON file will download automatically
8. Open the JSON file and copy the entire contents (you'll paste this into GitHub Secrets)

**Example JSON structure**:
```json
{
  "type": "service_account",
  "project_id": "your-project-id",
  "private_key_id": "...",
  "private_key": "-----BEGIN-----\n...\n-----END-----\n",
  "client_email": "...",
  "client_id": "...",
  "auth_uri": "https://accounts.google.com/o/oauth2/auth",
  "token_uri": "https://oauth2.googleapis.com/token",
  "auth_provider_x509_cert_url": "...",
  "client_x509_cert_url": "..."
}
```

### 1.2 Gmail Username (MAIL_USERNAME)

**What it is**: Your Gmail email address.

**Example**: `your-email@gmail.com`

### 1.3 Gmail App Password (MAIL_PASSWORD)

**What it is**: A 16-character app-specific password generated by Gmail (NOT your regular Gmail password).

**How to get it**:
1. Go to [myaccount.google.com](https://myaccount.google.com/)
2. Select **Security** from the left menu
3. Enable **2-Step Verification** (if not already enabled)
4. Go back to **Security** and scroll to **App passwords**
5. Select **Mail** and **Windows Computer** (or your device)
6. Google will generate a 16-character app password
7. Copy this password and paste directly into GitHub Secrets

**Important**: Use the app password, NOT your regular Gmail password. You cannot copy teh app password and paste somewhere else. You need to copy from the App Password Generator and paste directly into Github Secrets.

### 1.4 Collaborators Emails (COLLABORATORS_EMAILS)

**What it is**: One or more email addresses to receive workflow notifications.

**Example**: `you@example.com` or `email1@example.com,email2@example.com`

**Note**: Multiple emails should be comma-separated.

## Step 2: Add Secrets to GitHub

### 2.1 Navigate to Repository Settings

1. Go to your GitHub repository
2. Click the **Settings** tab at the top
3. In the left sidebar, click **Secrets and variables** → **Actions**

### 2.2 Create Secret: DBT_BIGQUERY_SERVICE_ACCOUNT_KEY

1. Click **New repository secret**
2. In the **Name** field, enter: `DBT_BIGQUERY_SERVICE_ACCOUNT_KEY`
3. In the **Secret** field, paste the entire contents of your BigQuery service account JSON file
4. Click **Add secret**

### 2.3 Create Secret: MAIL_USERNAME

1. Click **New repository secret**
2. In the **Name** field, enter: `MAIL_USERNAME`
3. In the **Secret** field, enter your Gmail address (e.g., `your-email@gmail.com`)
4. Click **Add secret**

### 2.4 Create Secret: MAIL_PASSWORD

1. Click **New repository secret**
2. In the **Name** field, enter: `MAIL_PASSWORD`
3. In the **Secret** field, paste your Gmail app password (the 16-character code)
4. Click **Add secret**

### 2.5 Create Secret: COLLABORATORS_EMAILS

1. Click **New repository secret**
2. In the **Name** field, enter: `COLLABORATORS_EMAILS`
3. In the **Secret** field, enter your email address(es) (e.g., `your-email@example.com`)
4. Click **Add secret**

---

## Step 3: Verify Secrets Were Created

1. Go to **Settings** → **Secrets and variables** → **Actions**
2. You should see all 4 secrets listed:
   - `DBT_BIGQUERY_SERVICE_ACCOUNT_KEY`
   - `MAIL_USERNAME`
   - `MAIL_PASSWORD`
   - `COLLABORATORS_EMAILS`

Each secret will show as masked (hidden) for security.

## Step 4: Test Locally (Optional)

Before running the GitHub Actions workflow, you can test locally with the same secrets:

```bash

conda env create -f environment.yml
conda activate hdb-elt

# Export secrets as environment variables

```

## Step 5: Run the Workflow

Once secrets are configured, you can trigger the workflow:

1. Go to **Actions** tab in GitHub
2. Select **Run ELT Process on Schedule** workflow
3. Click **Run workflow** → **Run workflow**
4. Monitor the workflow execution in real-time
5. Check the workflow logs for any errors

## Troubleshooting

### Secret not found or workflow fails
- Ensure all 4 secrets are created exactly as specified
- Check that secret names match exactly (case-sensitive)
- Verify the secret values are correct and complete

### BigQuery authentication fails
- Ensure the service account has BigQuery permissions (at minimum: `BigQuery Data Editor`, `BigQuery Job User`)
- Verify the GCP project ID in `hdb_resale_e2e/profiles.yml` matches your GCP project
- Ensure your GCP project is under `No Organization`

### Email notification fails
- Verify `MAIL_PASSWORD` is a Gmail app password, not your regular Gmail password
- Ensure 2-Step Verification is enabled on your Gmail account
- Check that `COLLABORATORS_EMAILS` contains at least one valid email address

### Workflow runs but produces errors
- Check the workflow logs on the **Actions** tab
- Review the email notification (if email is configured) for detailed error messages
- Test locally using the export commands to debug issues

## Security Best Practices

1. **Never commit secrets to GitHub** - Always use GitHub Secrets
2. **Rotate secrets regularly** - Update service account keys periodically
3. **Limit secret scope** - Only grant necessary permissions to service accounts
4. **Monitor secret usage** - Review workflow logs for unauthorized access attempts
5. **Use app passwords** - For Gmail, always use app-specific passwords, not your main password
